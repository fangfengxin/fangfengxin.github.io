什么是 JDBC ？

  • JDBC 概念：
    • Java DataBase Connectivity（Java 数据库连接）
    • 是 Java 访问关系型数据库的标准规范。
  • JDBC 本质：
    • 是官方（sun公司）定义的一套操作所有关系型数据库的规则，即接口。
    • 各个数据库厂商去实现这套接口，提供数据库驱动 jar 包。
    • 我们可以使用这套接口（JDBC）编程，真正执行的代码是驱动 jar 包中的实现类。

获取数据库连接

  • 参数：
    • url ：指定连接的路径
      • 语法：协议名:子协议://ip地址:端口号/数据库名称?参数=参数值 。
      • 如：jdbc:mysql://localhost:3306/dbdemo 。
      • 细节：如果连接是本机 MySQL 服务器，并且 MySQL 默认端口是 3306，则 url 可以简写为 jdbc:mysql:///数据库名称 。
      • 乱码的处理：指定参数为 ?characterEncoding=utf8，表示让数据库以 UTF-8 编码来处理数据。

PreparedStatement 接口

SQL 注入问题

  • SQL 注入问题：
    • 在拼接 SQL 时，有一些 SQL 的特殊关键字参与字符串的拼接，改变了原有 SQL 的真正意义。
    • 会造成安全性问题。
    • 如：在登录问题中，如果密码输入 a' or '1'='1 ，最终拼接成的 SQL 将变成 select * from user where name='newboy' and password='a' or '1'='1' ，错误的密码却登录成功。
  • 解决办法：
    • 使用 PreparedStatement 对象来解决。

预编译的 SQL

  • PreparedStatement 是 Statement 接口的子接口，继承于父接口中所有的方法。
  • 它是一个预编译的 SQL 语句：
    • SQL 的参数使用占位符 ? 。
    • 给 ? 赋值使用 setXxx(参数1, 参数2) 方法：
      • 参数 1 ：? 的位置编号。从 1 开始。
      • 参数 2 ：? 的值。

PreparedStatement 的执行原理

  • Statement 对象：
    • 每执行一条 SQL 语句，都会先将 SQL 语句发送到数据库，数据库先编译 SQL，再运行。
    • 如果有多条类似的 SQL 语句，数据库需要重复编译多次，再执行多次，效率低。
  • PreparedStatement 对象：
    • 先将 SQL 语句发送给数据库预编译，然后引用预编译后的结果，可以多次传入不同的参数给对象并执行。
    • 如果有多条类似的 SQL 语句，数据库只需要预编译 1 次，然后传入多次不同参数并执行，减少了 SQL 语句的编译次数，提高了执行效率。

注意事项

  • 后期都会使用 PreparedStatement 来完成增删改查的所有操作。
    1. 防止 SQL 注入，安全性更高。
    2. 有预编译的功能，执行效率更高。
    3. 提高了程序的可读性。

JDBC 管理事务

  • 事务：
    • 一个包含多个步骤的业务操作。如果这个业务操作被事务管理，则这多个步骤要么同时成功，要么同时失败。
  • 操作：
    1. 开启事务
    2. 提交事务
    3. 回滚事务
  • 使用 Connection 对象来管理事务：
    1. 开启事务：setAutoCommit(boolean autoCommit) ，调用该方法设置参数为 false，即开启事务。
    2. 提交事务：commit() ，当所有 SQL 都执行完提交事务。
    3. 回滚事务：rollback() ，在 catch 语句块中回滚事务。

数据库连接池

  • 数据库连接池：
    • 概念：
      • 其实就是一个存放数据库连接的容器（集合）。
    • 好处：
      1. 节约资源；
      2. 用户访问高效。
    • 实现：
      1. 标准接口 javax.sql.DataSource 。
        • 获取连接：getConnection()
        • 归还连接：Connection.close() ，如果连接对象 Connection 是从连接池中获取的，那么调用Connection.close() 方法，则不会再关闭连接了，而是归还连接。
      2. 一般不去实现它，由数据库厂商来实现。
        • C3P0
        • Druid（阿里巴巴提供的数据库连接池实现技术）

Druid 连接池

  • 使用步骤
    1. 导入 jar 包
    2. 定义配置文件：

Spring JDBC

  • Spring 框架对 JDBC 进行了简单封装，提供了一个 JdbcTemplate 对象简化 JDBC 的开发。
    • 创建 JdbcTemplate 对象依赖于数据源 DataSource。
    • 调用 JdbcTemplate 的方法来完成 CRUD 操作：
      • update() ：执行 DML语句（增删改）
      • queryForMap() ：查询结果将结果集封装为 Map 集合
        • 将列名作为 key，将值作为 value，将这条记录封装为一个 Map 集合
        • 注意：这个方法查询的结果集长度只能是 1
      • queryForList() ：查询结果将结果集封装为 List 集合
        • 注意：将每一条记录封装为一个 Map 集合，再将 Map 集合装载到 List 集合中
      • query() ：查询结果，将结果封装为 JavaBean 对象
        • 参数是 RowMapper ，一般我们使用 BeanPropertyRowMapper 实现类，可以完成数据到 JavaBean 的自动封装。
        • new BeanPropertyRowMapper<类型>(类型.class)
      • queryForObject ：查询结果，将结果封装为对象
        • 一般用于聚合函数的查询
